# 목동유쌤영어학원 2026 중등부 여름 무료특강 통합 신청 웹앱

초6~중3 대상 6개 무료특강을 한 페이지에서 비교하고, 필요한 수업을 여러 개 선택해 **한 번의 신청서로 Google Sheets에 접수**하는 모바일 중심 웹앱입니다.

- 프런트엔드: HTML · CSS · JavaScript 정적 웹페이지
- 배포: Vercel
- 신청 저장: Google Apps Script Web App
- 데이터 관리: Google Sheets
- 별도 유료 서버·DB: 필요 없음
- 기존 개별 Google Form: 데이터 참고용으로 코드에 보관되며 학부모 화면에는 노출하지 않음

## 2. 완성된 기능

### 학부모·학생 화면

- 실제 첨부 포스터 7종 적용
- 포스터 확대 보기
- 전체 / 선택한 특강 / 신청 가능 필터

- 최종 확인 화면
- 접수번호 자동 생성
- 중복 제출 방지용 요청 ID
- 선택 목록 24시간 임시 저장
- 모바일·태블릿·PC 반응형

### Google Sheets 관리자 화면

`setupSpreadsheet()`를 처음 한 번 실행하면 아래 시트가 자동 생성됩니다.

1. **신청내역**
   - 접수번호, 접수시각, 학생·보호자 정보, 선택 특강, 일정·선택 시간, 상담 메모 저장
   - 처리상태 드롭다운: 신규접수 / 상담중 / 상담완료 / 등록완료 / 대기 / 취소
   - 동일 요청 ID 중복 저장 차단

2. **특강별명단**
   - 한 번의 통합 신청을 선택 특강별로 한 행씩 자동 분리
   - 선생님별·특강별 필터링과 출석 명단 정리에 바로 활용
   - 특강별 처리상태 드롭다운 제공

3. **특강설정**
   - 교사명, 특강명, 대상, 일정, 수강료, 정원, 상태, 포스터, 소개문구 수정
   - `상태`를 `마감`으로 바꾸면 웹 선택 버튼 자동 비활성화
   - `정원`에 숫자를 입력하면 접수 시 정원 초과 방지
   - `세부선택지(|구분)`에 `오후 2시|오후 8시`처럼 입력하면 신청서에 선택 메뉴 생성

4. **환경설정**
   - 캠페인명, 대표전화, 신규 신청 알림 이메일 관리

# 4. Google Sheets 연동

웹 코드와 Apps Script 코드는 완성되어 있습니다. 다만 Google 계정 내부에 실제 Sheet를 만들고 권한을 승인하는 단계는 계정 소유자가 한 번 진행해야 합니다.

## 4-1. 새 Google Sheet 만들기

1. Google Drive에서 **새로 만들기 → Google 스프레드시트**를 누릅니다.
2. 파일 이름을 `2026 중등부 여름 무료특강 통합 신청내역` 등으로 지정합니다.
3. 상단 메뉴에서 **확장 프로그램 → Apps Script**를 누릅니다.

## 4-2. Apps Script 붙여넣기

1. Apps Script 편집기의 `Code.gs` 기존 내용을 모두 지웁니다.
2. 이 프로젝트의 `google-apps-script/Code.gs` 내용을 전부 복사해 붙여넣습니다.
3. 저장합니다.

`appsscript.json`은 선택 사항입니다. `Code.gs`만으로도 동작합니다.

## 4-3. 시트 자동 생성

1. Apps Script 상단 함수 선택 메뉴에서 `setupSpreadsheet`를 선택합니다.
2. **실행**을 누릅니다.
3. 처음 실행 시 Google 권한을 승인합니다.
4. Sheet로 돌아가 아래 4개 탭이 생성됐는지 확인합니다.
   - 신청내역
   - 특강별명단
   - 특강설정
   - 환경설정

특강설정 값을 처음 상태로 다시 돌려야 할 때만 `resetCourseSettingsToDefaults`를 실행합니다. 이 함수는 현재 특강설정 값을 첨부 포스터 기준 기본값으로 덮어씁니다.

## 4-4. 웹 앱 배포

1. Apps Script 우측 상단 **배포 → 새 배포**
2. 유형 선택에서 **웹 앱**
3. 설정
   - 설명: `2026 중등부 여름 무료특강 통합 신청서`
   - 다음 사용자로 실행: **나**
   - 액세스 권한: **모든 사용자**
4. **배포**
5. `https://script.google.com/macros/s/.../exec` 형태의 웹 앱 URL 복사

Apps Script 코드를 나중에 수정했다면 **배포 → 배포 관리 → 수정 → 새 버전 → 배포**를 해야 실제 웹 앱에 반영됩니다.

## 4-5. 웹페이지와 연결

`config.js`에서 아래 값만 교체합니다.

```js
googleScriptUrl: 'PASTE_GOOGLE_APPS_SCRIPT_WEB_APP_URL_HERE',
```

예시:

```js
googleScriptUrl: 'https://script.google.com/macros/s/AKfycbxxxxxxxx/exec',
```

이후 Vercel에 다시 배포하면 신청서가 Google Sheets에 저장됩니다.

# 9. 토모 일정 슬랙 자동 알림

구글 캘린더에 **제목이 `토모-xxx` 형식**인 일정이 잡혀 있는 날, **당일 오전 10시경** 지정한 슬랙 채널로 아래 문구를 자동 발송하는 기능입니다. (오늘 토모 일정이 없으면 아무 것도 보내지 않습니다.)

```text
[토모 관리 유의 사항]
아래 예시에 해당하는 경우, 이 글에 댓글 달아서 보고하세요.

• 토모 시작 후 듣기를 5분이 지나서 튼 경우
• 스피커 연결이 안 되어서 딜레이 된 경우
• 중간에 학생이 퇴실한 경우
• 시작 시간 이후에 학생이 입실한 경우
• 한 번호로 찍고 자는 학생 등등..
```

코드는 `google-apps-script/TomoSlackNotifier.gs`에 있으며, 기존 신청서 Apps Script와 **같은 프로젝트에 함께 넣으면** 됩니다. 캘린더 iCal 주소·슬랙 토큰 같은 비밀값은 저장소에 커밋하지 않고 Apps Script의 **Script Properties**에 저장합니다.

## 9-1. 슬랙 채널 준비 (둘 중 하나)

- **(권장) Incoming Webhook** — 슬랙 워크스페이스에서 알림을 받을 채널용 Webhook URL을 발급받습니다. 채널이 Webhook에 고정됩니다.
  `https://api.slack.com/messaging/webhooks` 참고. 형식: `https://hooks.slack.com/services/...`
- **Bot Token** — `chat:write` 권한이 있는 봇 토큰(`xoxb-...`)을 발급받고, 알림 채널에 봇을 초대한 뒤 채널명(`#채널명`) 또는 채널 ID를 사용합니다.

## 9-2. 설치 (최초 1회)

1. Apps Script 편집기에서 **파일 추가 → 스크립트**로 `TomoSlackNotifier.gs`를 만들고 이 파일 내용을 붙여넣습니다.
2. `setupTomoNotifier()` 함수 안의 `config`에 값을 채웁니다.
   - `icalUrl`: 구글 캘린더 iCal 비공개 주소(`...basic.ics`)
   - `slackWebhookUrl`: Incoming Webhook URL (Bot Token을 쓰면 비워 두고 아래 두 값 입력)
   - `slackBotToken` / `slackChannel`: Bot Token 방식일 때만 입력
3. 함수 선택 메뉴에서 `setupTomoNotifier`를 선택하고 **실행**합니다.
   - 값이 Script Properties에 저장되고, **매일 오전 10시 트리거**가 자동 설치됩니다.
   - 처음 실행 시 캘린더 조회(UrlFetch)·트리거 권한을 승인합니다.
4. 저장이 끝나면 코드에 적어둔 `config` 값은 지워도 됩니다. **저장소에는 커밋하지 마세요.**
5. `testTomoDailyReminder`를 실행해 오늘 토모 일정이 있을 때 슬랙으로 실제 발송되는지 확인합니다.

## 9-3. 확인·운영 함수

| 함수 | 용도 |
|---|---|
| `setupTomoNotifier` | iCal·슬랙 정보 저장 + 매일 오전 10시 트리거 설치 |
| `sendTomoDailyReminder` | 트리거가 매일 호출(수동 실행도 가능) |
| `testTomoDailyReminder` | 지금 실행해 실제 발송 테스트 |
| `previewTomoToday` | 오늘 감지되는 토모 일정만 로그로 확인(발송 안 함) |
| `showTomoStatus` | 저장된 설정·트리거 상태 확인(민감정보 마스킹) |
| `disableTomoNotifier` | 알림 트리거 제거(저장값은 유지) |

## 9-4. 참고 사항

- 시간대는 `Asia/Seoul` 기준입니다. 종일 일정·시간 지정 일정 모두 지원합니다.
- 매주/매일 반복 일정(RRULE)과 예외 날짜(EXDATE), 개별 수정 일정(RECURRENCE-ID)을 반영해 "오늘 실제 발생하는" 토모 일정만 감지합니다.
- Apps Script 시간 트리거는 지정 시각의 1시간 이내(예: 10:00~11:00)에 실행됩니다.
- 제목 매칭은 `토모-`로 시작하는 일정입니다(하이픈 `-`·`–`·`—` 및 앞뒤 공백 허용).
